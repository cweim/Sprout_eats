**Whisper capabilities (from research):**
- `model.transcribe()` returns dict with "text" and "language" keys
- `task="translate"` translates any language to English
- For multilingual models (base, small, medium, large), translation works
- Language is auto-detected and returned as ISO code (e.g., "en", "ja", "ko")

**Current transcriber:** Returns only text string, no language info

<tasks>

<task type="auto">
  <name>Task 1: Create TranscriptionResult dataclass and update transcribe_audio</name>
  <files>services/transcriber.py</files>
  <action>
1. Create TranscriptionResult dataclass with fields:
   - text: str (original transcription in detected language)
   - language: str (ISO language code, e.g., "en", "ja")
   - english_text: str | None (English translation if not already English)

2. Update transcribe_audio() to:
   - First transcribe with task="transcribe" to get original text + language
   - If detected language != "en", run again with task="translate" to get english_text
   - Return TranscriptionResult

3. Handle edge case: If language is "en", set english_text = text (no second call needed)

Use model.transcribe(str(audio_path), task="transcribe") first,
then model.transcribe(str(audio_path), task="translate") if needed.

Note: This means two Whisper calls for non-English content. Accept this tradeoff
for accuracy (translate task produces better English than trying to search with
transliterated non-English text).
  </action>
  <verify>
Manual test:
- transcribe_audio() returns TranscriptionResult dataclass
- For English audio: language="en", english_text == text
- For non-English audio: language detected, english_text is English translation
  </verify>
  <done>TranscriptionResult returned with language detection and optional English translation</done>
</task>
