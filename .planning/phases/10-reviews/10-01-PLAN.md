<task type="auto">
  <name>Task 3: Add API endpoints for reviews</name>
  <files>api/routes.py</files>
  <action>
Add new endpoints to the FastAPI router:

1. GET /api/places/{place_id}/review
   - Returns review for place with dishes and photos
   - Response includes nested dishes with their photos
   - 404 if no review exists
   - Response schema matches PLAN.md Review Response Schema

2. POST /api/places/{place_id}/review
   - Create or update review for place
   - Request body: overall_rating, price_rating, overall_remarks, dishes[]
   - Each dish: id (null for new), name, rating, remarks
   - Returns created/updated review
   - Validates: ratings 1-5, dish name required, dish rating required

3. DELETE /api/places/{place_id}/review
   - Delete review for place
   - Returns 204 No Content
   - 404 if no review exists

4. GET /api/reviews
   - Returns all reviews for current user
   - Includes place name in response for display
   - Ordered by updated_at desc

5. POST /api/reviews/{review_id}/photos
   - Upload photo to review
   - Form data: file (image), dish_id (optional)
   - Send image to Telegram Bot API to get file_id
   - Store file_id in review_photos
   - Check photo limits (max 2 per dish, max 3 overall)
   - Return photo object with id and url

6. DELETE /api/reviews/{review_id}/photos/{photo_id}
   - Delete photo from review
   - Returns 204 No Content

Add Pydantic models for request/response validation:
- ReviewRequest, ReviewResponse, DishRequest, DishResponse, PhotoResponse
